## TOC Format Standardization

All TOCs follow a consistent format:

### Standard Format

```markdown
## Table of Contents

- [Section Title](#section-title)
  - [Subsection Title](#subsection-title)
- [Another Section](#another-section)
```

### Formatting Rules Applied

1. **H2 Headings:** Listed as top-level items with `-` bullet points
2. **H3 Headings:** Indented with 2 spaces and listed under their parent H2
3. **Anchor Links:** 
   - Converted to lowercase
   - Spaces replaced with hyphens
   - Special characters removed (emojis preserved in display but not in anchor)
   - Numbers preserved (e.g., "1. Overview" â†’ "#1-overview")
4. **Separator:** Horizontal rule (`---`) added after TOC and before first content section
5. **Consistency:** All TOCs placed immediately after the document title/description

### Special Cases Handled

- **Numbered Sections:** Sections like "1. Overview" are converted to anchor `#1-overview`
- **Emoji Prefixes:** Emojis in headings (like "ðŸ§© Node.js API Backend") are preserved in the TOC text but removed from anchor links
- **Parentheses:** Content in parentheses (like "(Optional)") is preserved in both TOC text and anchor links
- **Special Characters:** Colons, periods, and other punctuation are handled appropriately in anchor generation
